TikTok Live Reader

Add-on ini terhubung ke siaran TikTok Live dan mengekstrak informasi secara real-time. Add-on menggunakan library Python TikTokLive untuk mengakses siaran live tertentu. Fitur-fitur berikut didukung:
  • Menyimpan data hasil ekstraksi ke dalam file teks, yang dibuat di folder Documents\TikTok live.
  • Melacak peristiwa: komentar, pengikut baru, hadiah (gift), likes, share, pengunjung, dan permintaan tamu.
  • File events yang dapat merekam berbagai jenis data secara bersamaan.
  • Notifikasi audio untuk peristiwa terpilih, seperti komentar baru, hadiah, dll.
  • Statistik siaran: menampilkan jumlah penonton saat ini, total pengunjung sesi, total likes, pengikut baru, dan diamond yang terkumpul.
  • Peringkat Top Gifters: mengurutkan pengguna dari yang paling aktif hingga yang paling sedikit berdasarkan diamond yang dikirim.
  • Peringkat Top Likers: mengurutkan pengguna berdasarkan total likes selama siaran.
TikTok Live Reader membantu host atau penonton memantau data penting siaran tanpa alat eksternal. Data disimpan dalam file teks yang terletak di folder Documents\TikTok live dan dapat ditinjau kembali nanti. Add-on mengekstrak tipe data berikut ke dalam file terpisah:
  • comments.txt - Mencatat semua komentar yang terdeteksi selama live beserta timestamp.
  • followers.txt - Mencatat semua pengguna baru yang mengikuti kreator selama sesi berjalan.
  • gifts.txt - Mencatat semua hadiah yang diterima: pengguna, jenis hadiah, jumlah, dan timestamp.
  • likes.txt - Menunjukkan siapa yang memberi like, kapan, dan berapa kali.
  • requests.txt - Mencatat semua permintaan bergabung yang dikirimkan kepada host.
  • shares.txt - Mencatat pengguna yang membagikan siaran live beserta timestamp.
  • stats.txt - Menampilkan penonton saat ini, pengikut baru, total likes, pengunjung sesi, dan total diamond.
  • top gifts.txt - Peringkat pengguna yang mengirim hadiah, dari yang paling aktif sampai paling sedikit.
  • top likes.txt - Peringkat pengguna berdasarkan total likes.
  • visitors.txt - Mencatat semua pengunjung yang masuk selama sesi berjalan beserta timestamp.
  • events.txt - Terletak di antara file komentar dan pengikut selama navigasi keyboard, file ini mencatat berbagai jenis peristiwa yang ditentukan dalam pengaturan. Tujuannya adalah untuk melacak komentar, pengikut, hadiah, likes, permintaan, share, atau pengunjung dengan mudah di satu tempat.

Pengaturan

TikTok Live Reader memiliki jendela pengaturan sendiri, yang dapat dibuka dengan pintasan NVDA+Shift+Control+L. Jendela ini memiliki 4 tab:

General

Bagian ini mencakup pengaturan berikut:
  • Username - Masukkan username host TikTok tanpa simbol @.
  • Clear files on startup - Menghapus isi file teks saat add-on dimulai.
  • Clean usernames - Opsi ini menghapus elemen yang tidak perlu dari nama pengguna. Banyak pengguna TikTok menggunakan emoji dan simbol Unicode yang menghambat screen reader. Saat aktif, nama hanya akan berisi huruf, angka, dan tanda baca.
  • Connection retry count - Menentukan berapa kali add-on mencoba tersambung ke siaran live.

Events

Bagian ini berisi kotak centang untuk menentukan data mana yang dicatat di events.txt (Comments, Followers, Gifts, Likes, Requests, Shares, dan Visitors).

Auto speak

Bagian ini menentukan data mana yang diumumkan saat fungsi Auto speak aktif. Aktifkan dengan NVDA+Shift+Control+S atau kotak centang "Auto speak selected events". Saat aktif, peristiwa tertentu dapat dipilih untuk dibacakan otomatis (misalnya "New follower: Username").

Sounds

Bagian ini menentukan peristiwa live mana yang akan memicu isyarat audio pendek. Agar fitur ini berfungsi, Anda harus mengaktifkan opsi "Play sounds for the selected events" terlebih dahulu. Setelah diaktifkan, kotak centang akan muncul, disusun dalam urutan yang persis sama dengan dua bagian sebelumnya. Anda juga dapat mengaktifkan/menonaktifkan suara menggunakan pintasan NVDA+Shift+Control+P. Opsi berikut juga tersedia:
  • Volume - Penggeser ini menentukan tingkat kekerasan suara notifikasi peristiwa.
  • Learn sounds - Memutar deskripsi suara agar pengguna dapat mendengarnya terlebih dahulu.
Suara dan pesan diputar dengan jeda satu detik dan disimpan dalam file sementara untuk memastikan pelaporan tetap berurutan saat traffic live tinggi. Fungsi suara dan Auto speak bekerja secara independen.

Cara menggunakan add-on

Tekan NVDA+Shift+Control+T dan klien TikTok Live Reader akan mencoba terhubung. Status koneksi diumumkan otomatis. Setelah terhubung, hotkey berikut tersedia:
  • Control+Shift+Panah Kanan - File berikutnya (misalnya Comments, Events, Followers, dll).
  • Control+Shift+Panah Kiri - File sebelumnya.
  • Control+Shift+Panah Bawah - Item berikutnya di file saat ini (misalnya komentar, like, share).
  • Control+Shift+Panah Atas - Item sebelumnya.
  • Control+Shift+Home - Pilih item pertama di file saat ini.
  • Control+Shift+End - Pilih item terakhir.
  • NVDA+Shift+Control+V - Melaporkan jumlah penonton saat ini.
  • NVDA+Shift+Control+S - Mengaktifkan atau menonaktifkan Auto speak.
  • NVDA+Shift+Control+P - Mengaktifkan atau menonaktifkan suara.
  • NVDA+Shift+Control+R - Menghapus isi file teks dan file sementara.
Tekan NVDA+Shift+Control+T untuk mengakhiri sesi add-on.

Catatan

  • Membuka pengaturan menonaktifkan sementara auto-speak dan suara, tetapi pencatatan data tetap berlanjut di latar belakang.
  • Jika username diubah di pengaturan, add-on akan mencoba terhubung ke pengguna baru setelah disimpan.
  • Saat pembersihan file otomatis dinonaktifkan, file akan tetap berisi data dari siaran live terakhir. Jika Anda ingin menghapusnya di awal sesi saat ini, tekan NVDA+Shift+Control+R.
  • Jika pembersihan manual dinonaktifkan, add-on mengingat posisi terakhir Anda di file (misalnya baris 45 di komentar).

Batasan

  • Daftar penonton lengkap tidak dapat ditampilkan karena library TikTokLive tidak melaporkan saat pengguna keluar.
  • Peringkat berdasarkan durasi tonton tidak dapat dibuat karena alasan yang sama.
  • Nama hadiah dilaporkan dalam bahasa Inggris karena jumlahnya terlalu banyak dan terlalu sering diperbarui untuk diterjemahkan secara layak.
  • Permintaan bergabung hanya didaftarkan jika pengguna telah berinteraksi dengan siaran live sebelumnya. Ini karena TikTok tidak mempublikasikan nama pengguna yang hanya mengirim permintaan tanpa aktivitas lain.

Daftar Perubahan

Versi 1.6

  1. Add-on sekarang kompatibel dengan NVDA 2026.1.
  2. Pengaturan di bagian Events sekarang hanya berlaku untuk file events.txt. Ini menentukan jenis data apa saja yang akan dicatat di dalamnya.
  3. Bagian Sounds telah ditambahkan, dengan opsi pengalih utama "Play sounds for the selected events".
  4. Sebagai hasilnya, fungsi bicara otomatis (auto-speak), pemutaran suara, dan perekaman data dalam file events bekerja secara independen satu sama lain.

Versi 1.5

  1. Logika untuk memuat dan mematikan add-on telah distabilkan.
  2. Migrasi ke NVDA addon template untuk otomatisasi dan lokalisasi yang lebih baik.
  3. Logika pencatatan untuk likes, share, dan follows telah distabilkan.
  4. Opsi Auto speak sekarang dipatuhi; pengumuman dimulai 10 detik setelah koneksi berhasil.
  5. Tab Auto speak sekarang secara dinamis menyembunyikan pilihan peristiwa kecuali fungsi utama diaktifkan.
  6. Memperbaiki bug di mana suara dan pengumuman dimainkan segera setelah restart add-on.
  7. Menambahkan pintasan NVDA+Shift+Control+P untuk mengaktifkan/menonaktifkan suara selama sesi aktif.
  8. Menambahkan tombol akses (Alt + huruf) untuk semua pengaturan.
  9. Logika slider Audio volume dioptimalkan. Labelnya sekarang adalah Volume.
  10. Pengaturan sekarang disimpan secara dinamis. Pencatatan latar belakang berlanjut saat jendela pengaturan terbuka.
  11. Koneksi otomatis ke pengguna baru jika diubah dalam sesi aktif.
  12. Menambahkan dukungan untuk melacak permintaan bergabung. Disimpan dalam file requests.txt.
  13. Opsi "Requests" baru di pengaturan dan notifikasi suara baru untuk peristiwa tersebut.
  14. Menambahkan validasi input username: minimal dua karakter dan hanya boleh berisi huruf Latin, angka, titik, dan garis bawah.
